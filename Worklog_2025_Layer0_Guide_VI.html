📋 Worklog — Layer 0

Hướng dẫn tạo Input

Từ thiết lập All_Bug_Report đến Refresh All_Input.

00
Thiết lập All_Bug_Report
Làm 1 lần duy nhất khi thêm project mới
📊 Block trái — Data hàng ngày
A1 = Project  |  B1 = Date  |  C1 trở đi = tên từng member (mỗi member 1 cột, liên tục — không để trống cột giữa)

Tên member phải khớp chính xác với REF_MEMBER. Mỗi dòng = 1 project × 1 ngày × số bug từng member. Data được kéo tự động từ file project qua IMPORTRANGE.
📋 Block phải — Danh sách project
Điền: Mã project, Tên project, SpreadsheetId của file Google Sheet project tương ứng.

Dùng công thức IMPORTRANGE("SpreadsheetId", "Workload!A1") để kéo data từ file project về. Mỗi file project cần có sheet Workload đúng format.

Lần đầu dùng IMPORTRANGE phải click Allow accessđể cấp quyền truy cập.

Sau đó bấm nút [ Update All Bug Report] ở sheet All_Bug_Report
💡 Sau khi thiết lập xong, block trái tự cập nhật realtime khi file project có bug mới — không cần làm lại.
↓
01
Tạo Input Sheet
Chạy mỗi đầu tháng

Set Config!B1 = tháng  |  Config!B2 = năm  →  Click nút ▶ Tạo Input

Script tạo sheet Input_xx với toàn bộ ngày thứ 2→6 trong tháng và member active, điền sẵn cột Date + Member.

⚠ Lưu ý: Nếu Input_xx đã tồn tại, script sẽ tạo sheet mới tên Input_xx(1), Input_xx(2)... thay vì ghi đè. Kiểm tra Config!B3 sau khi tạo để chọn đúng sheet cần dùng.
↓
02
Refresh Danh Sách Sheet
Chạy sau khi tạo Input mới

Click nút ▶ Refresh Sheet List

Cập nhật dropdown tại Config!B3 với danh sách tất cả Input_* sheets hiện có trong file. Chọn sheet cần làm việc trước khi qua bước tiếp theo.

↓
03
Import Bug Report
Kéo bug từ All_Bug_Report vào Input

Chọn sheet target trong dropdown Config!B3  →  Click nút ▶ Import Bug Report

Script đọc All_Bug_Report và tự động điền Bug Report vào Input sheet đã chọn:

✅ Dòng đã có (cùng Date × Member × Project) → cập nhật Output_Count
➕ Dòng chưa có → thêm dòng mới vào cuối
🔢 Dòng trùng → xóa duplicate, giữ dòng đầu tiên
↓
04
Điền Tay Phần Còn Lại
Thực hiện trong Input sheet

Điền thủ công các cột còn thiếu cho từng dòng:

J — Day Type  →  Full / Half / Off / Holiday cho mỗi ngày
D — Project + E — Task + F — Time Spent + G — Output_Type + H — Output_Count  →  cho các task không phải Bug Report
💡 Tham khảo Input Guidelines để biết quy định điền từng cột.
↓
05
Kiểm Tra Bug Report
Chạy để xác nhận dữ liệu trước khi submit

Click nút ▶ Check Bug Report

So sánh Bug Report giữa All_Bug_Report và tất cả Input_* sheets theo năm tại Config!B2. Kết quả ghi vào Config cột J:Z gồm 3 nhóm:

🔴 Missing — Có trong All_Bug_Report nhưng chưa điền vào Input
🟡 Extra — Có trong Input nhưng không có trong All_Bug_Report
🟠 Mismatch — Số lượng bug không khớp giữa 2 nguồn
⚠ Lưu ý: Script chỉ đọc, không sửa dữ liệu. Nếu có lỗi, quay lại Bước 3 hoặc điền tay để sửa.
↓
06
Refresh All Input
Bước cuối — gom tất cả vào All_Input

Click nút ▶ Refresh All Input

Gom tất cả Input_* sheets vào All_Input. Đây là nguồn dữ liệu cho toàn bộ pipeline Layer 1 trở lên.

💡 Chạy lại bước này mỗi khi có thay đổi trong bất kỳ Input_* sheet nào.
